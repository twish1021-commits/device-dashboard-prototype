# 设备看板项目规则

## 项目定位
- 本项目是设备看板原型项目。
- 当前主原型文件为 `arcade_dashboard_v3.html`。
- 需求说明与原型说明分别保存在 `设备看板需求文档.txt`、`设备看板原型设计描述.txt`。

## 工作方式
- 默认中文沟通。
- 修改前先阅读需求文档和原型说明。
- 修改原型时优先保持单文件 HTML 原型可直接打开运行。
- 资源文件统一放在 `assets/` 目录下。

## 文件约定
- 页面原型文件使用 `.html`。
- 文档文件名使用中文。
- 新增图片、图标等静态资源放入 `assets/` 下按用途分目录。
- 不随意新增顶层目录。

## 编码约定
- 新增和修改的文本文件统一使用 UTF-8。
- HTML 文件必须声明 `<meta charset="UTF-8">`。
- 如发现旧文件乱码，先确认编码原因，再进行修复。

## 原型质量要求
- 页面第一屏必须直接呈现可操作看板，不做营销页。
- 交互说明应覆盖：触发条件、执行动作、反馈状态。
- 涉及离线、无数据、刷新间隔不足、报警等异常状态必须保留。
- 涉及多门店独立部署的数据，仅做当前门店视角展示；跨门店数据不得假设实时一致。

## 验证要求
- 改完必须打开页面验证。
- 视觉改动需要检查桌面视口下是否有乱码、遮挡、溢出和明显错位。
- 交互改动需要至少验证相关点击路径。

## 预览与发布
- 当前线上预览地址为 `https://twish1021-commits.github.io/device-dashboard-prototype/`。
- GitHub 仓库为 `twish1021-commits/device-dashboard-prototype`。
- 当前长期分享方式为 GitHub Pages 静态发布。
- 本地原型仍以 `arcade_dashboard_v3.html` 为主文件。
- `index.html` 是用于静态站点发布的入口文件，修改主原型后需要同步更新。
- `vercel-deploy/` 是历史 Vercel 部署目录，Vercel 在国内访问不稳定，后续默认不作为主要交付链接。
- Codex 内置浏览器优先打开 GitHub Pages 地址进行标注；`file://` 和 `127.0.0.1` 本地地址可能被当前会话安全策略拦截。

## 上下文沉淀
- 每轮原型修改完成后，必须同步更新 `设备看板需求文档.txt` 和 `设备看板原型设计描述.txt` 中对应内容。
- 聊天中的临时结论如果会影响后续协作，应写入本文件或需求/原型说明文档，不只保留在聊天记录里。
- 重开对话时，先读取本文件、需求文档、原型说明和当前 HTML，再继续工作。
- 对外分享链接、发布方式、主文件关系发生变化时，优先更新本文件。

## 当前原型关键约定
- 后台菜单壳按世软科技系统样式展示。
- 一级菜单当前选中【设备】。
- 【设备看板】作为独立二级菜单显示在【扣费设备】分组上方。
- 右侧工作区以系统标签页方式打开【设备看板】。
- 顶部系统栏显示切换系统、搜索、审批、下载、知识库、更多、版本、用户信息。
- 二级菜单上方品牌区域只显示【世软科技】，不显示切换系统按钮。
- 报表弹窗必须保留查询条件区、查询按钮、导出按钮。
- 报表只隐藏需求指定的查询条件，其余查询条件必须显示。
- 未处理警报只统计并展示游戏机未处理异常。
- 未处理警报弹窗只显示游戏机警报，不显示库存、自助机标签页。
- 游戏机单机卡片显示 P 位数量，格式为 `P位数:2`。
- 机种卡片不显示 P 位数量。
- 所有界面不显示【当日实时数据】文字。
- 全部页签设备标题不显示【区块】文字。
- 游戏机页签按类型视图不显示顶部总数量条，在各分类标题中显示对应机台数量。
- 游戏机按类型分类标题不显示英文状态标签。
- 离线设备分组名称显示收银台、自助机、游戏机、大项目，不显示【断连】。
- 彩票机、礼品机卡片尺寸必须保持一致，切换数据视图时不能遮挡指标。
- 本地修改 `arcade_dashboard_v3.html` 后必须同步更新 `index.html`。
